1. Umumiy ma'lumot
Auditor - tashkilotlarda axborot xavfsizligi auditini avtomatlashtirilgan tarzda rejalashtirish, o'tkazish, hujjatlashtirish va hisobotlash uchun mo'ljallangan korporativ web-platforma. Tizim audit obyektlari ma'lumotlar bazasini, auditorlar guruhini, audit jarayonlarini, topilmalarni (zaifliklarni), mutaxassislar reytingini (KPI) va barcha topilmalar/loyihalarning ko'p bosqichli tasdiqlash oqimini markazlashtirilgan tarzda boshqaradi.
Hujjatning tayinlanishi
Ushbu Texnik Topshiriq tizimning amalga oshirilgan (joriy) holatini kodbaza tahlili asosida to'liq aks ettiradi. Hujjat yangi modullarni yoki keyingi versiyalarni ishlab chiqish uchun me'yoriy baza, shuningdek tizimni qabul qilish va texnik hujjatlashtirish uchun asos vazifasini bajaradi.
2. Loyihaning maqsadi va vazifalari
2.1. Asosiy maqsad
Kiberxavfsizlik auditini birxillashtirilgan jarayon asosida amalga oshirish, barcha auditorlar ishini bitta platformada boshqarish, audit ma'lumotlarining to'liqligi, sifati va izlanuvchanligini (traceability) ta'minlash.
2.2. Asosiy vazifalar
Audit obyektlarini (tashkilotlar, kontaktlar, qurilma inventarini) markazlashtirilgan boshqarish
Auditorlar guruhini shakllantirish va vazifalarni taqsimlash
Audit loyihasini (project) 3-bosqichli tasdiqlash oqimi orqali rasmiylashtirish
Topilgan zaifliklarni 3-bosqichli tasdiqlash orqali rasmiy holatga keltirish
Tahlil natijalarini avtomatlashtirilgan parserlar (config / scanner / traffic) orqali import qilish
AI (mahalliy LLM) yordamida tavsiya va hisobotlarni boyitish
Yakuniy hisobotlarni PDF / DOCX / XLSX / HTML formatlarida eksport qilish
Mutaxassislar samaradorligini KPI ball tizimi orqali baholash
Barcha amallarning audit_logs jurnali orqali izlanuvchanligini ta'minlash
EXE Desktop Agent orqali offline ishlash va ma'lumotlarni server bilan sinxronlash

Defense-in-depth: UI (PermissionGuard) -&gt; pipeline (AuthorizationBehavior) -&gt; handler (ownership/duty) -&gt; DB (RLS + check constraint)

Umumiy build sozlamalari Directory.Build.props faylida: Nullable=enable, ImplicitUsings=enable, TreatWarningsAsErrors=true, AnalysisLevel=latest-recommended, StyleCop + SonarAnalyzer. Default til (NeutralLanguage): uz.

4.5. Avtorizatsiya pipeline'i
Har bir command/query MediatR pipeline orqali quyidagi tartibda o'tadi:
ValidationBehavior - FluentValidation orqali kirish ma'lumotlarini tekshirish
AuthorizationBehavior - CommandPermissions.Map'dan kerakli ruxsatlar olinadi; foydalanuvchida ulardan KAMIDA BITTASI (OR mantiq) yoki super ('*') bo'lishi shart
Handler - biznes logikasi, ko'pincha qo'shimcha duty/ownership (egalik) tekshiruvi bilan (defense-in-depth)
Misol: UpdateTaskStatusCommand CommandPermissions'da yo'q - ruxsat handler ichida egalik bo'yicha tekshiriladi (assignee | group_lead duty | super). Bu DB darajasidagi RLS (Row Level Security) bilan birga ko'p qatlamli himoyani ta'minlaydi.

5. Domen modeli
DbContext (AuditorDbContext) ~38 ta DbSet'dan iborat. Asosiy entity'lar agregatlar bo'yicha:
5.1. Identity
User - foydalanuvchi (username, email, full_name, role, department, position, 2FA, lockout holati)
Role - system rol (5 ta seed: super/head/chief/lead/t1, daraja bilan)
Permission - ruxsatlar katalogi (modul.amal)
PasswordResetToken - parol tiklash tokenlari
LoginAttempt - login urinishlar (success/fail, IP, User-Agent)
5.2. Organizations
Organization - tashkilot (code, localized name, TIN, manzil, soha, risk darajasi)
OrganizationContact - tashkilot kontaktlari
OrganizationDevice - tashkilot qurilmalari inventari
5.3. Audits
Audit - audit obyekti (code, name, organization, audit_type, start/end date, status, progress)
AuditGroup - audit guruhi (audit'ga 1:1)
AuditGroupMember - guruh a'zolari (user, role_snapshot, primary_duty)
AuditProject - audit loyihasi (goal, scope, methodology, targets, tools, approval stage)
AuditProjectApproval - loyiha tasdiqlash timeline (o'zgarmas/immutable)
5.4. Tasks
AuditTask - vazifa (code, name, task_type, priority, due_date, assignee, status, progress)
TaskAssignment - vazifa biriktirish tarixi
TaskStatusHistory - vazifa status o'zgarishlari jurnali
5.5. Vulnerabilities / Findings
Vulnerability - topilma (code, title, severity, CVSS, CWE, CVE, status, current_approval_stage, idempotency_key)
VulnerabilityEvidence - dalil fayllar (screenshot, log, config, scan, traffic, document)
VulnerabilityApproval - topilma tasdiqlash timeline (immutable: stage, action, actor, comment)
CvssScore - CVSS v3.1 metrikalari (vector + score -&gt; severity)
5.6. Analysis
ConfigAnalysisFile - qurilma konfiguratsiyasi (Cisco / Linux SSH / sudoers / Nginx / Apache / MikroTik)
ScannerImport - skaner natijalari (Nessus / Nmap / OpenVAS / Burp / ZAP / IP-scanner)

5.7. Reports
Report - hisobot (audit'ga 1:N, status: draft/approved, version, classification)

5.8. KPI

5.9. Agent (12 ta jadval)
